# Flowrium Voice 文档编写设计

## 背景

Flowrium Voice 是一个基于 FunASR 的公司级语音转文字（STT）快速验证原型。项目目前只有一个 commit，包含：
- `inference/main.py` — FastAPI 网关服务
- `web-demo/index.html` — 浏览器端演示页面
- `README.md` — 基础快速开始指南

现有文档不足以支撑团队理解项目全貌和向管理层汇报验证结果。

## 目标

1. 让开发团队快速理解项目架构、设计决策和部署方式
2. 向管理层提供结构化的 STT 性能验证报告
3. 降低新成员上手成本

## 受众

- **开发团队**：需要技术细节、架构说明、部署运维指南
- **管理层**：需要验证结果摘要、性能指标、可行性结论

## 语言

全部中文编写。

## 方案：README 扩展 + docs/validation-report.md

选择单文件扩展方案，原因：项目当前规模小（2 个源文件），单文件扩展最实用。验证报告单独放 `docs/` 方便后续更新测试数据。

### README.md 重构结构

1. **项目简介** — 一句话说明 Flowrium Voice 是什么、做什么
2. **技术架构** — 保留现有架构图，增加：
   - 设计决策：为什么选 FunASR
   - 两种连接模式：浏览器直连 vs gateway 模式
   - gateway 模式的价值（统一入口、后续扩展）
3. **快速开始** — 保留现有 Docker + Web Demo 步骤，补充 gateway 模式启动
4. **组件说明** — `inference/` 和 `web-demo/` 的功能描述
5. **部署运维** — 新增：
   - Docker 配置详情（端口、资源限制）
   - 端口说明（10095/10096/8000）
   - 常见问题排查（FunASR 连不上、麦克风权限、音频格式）
6. **验证清单** — 保留现有 checklist，扩展测试方法

### docs/validation-report.md 结构

1. **报告概述** — 验证目标、测试环境（硬件、OS、Docker 版本）
2. **测试方法** — 每个测试项的测试方法说明
3. **测试结果** — 表格形式，按项填写：
   - 中文转写精度
   - 英文转写精度
   - 中英混合（code-switching）
   - 实时延迟（2pass partial/final）
   - 办公噪声鲁棒性（安静/风扇/键盘/人声）
   - CPU 负载（空闲/单路/多路并发）
4. **结论与建议** — 是否满足公司级 STT 需求，后续建议

数据部分用 `[待测试]` 占位，方便后续填入实际测试数据。

## 不做的事

- 不写 API 参考文档（当前只有 2 个端点，README 中已足够说明）
- 不建独立的 docs/ 站点（项目规模不需要）
- 不写贡献指南（内部项目，不需要）
