# Flowrium Voice - 语音转写验证计划

## 产品方向

大屏语音交互系统。校长/教师通过语音操作大屏，查看学校统计面板信息、处理日常事务。

## 背景

### 上次原型复盘

2026-05-18 完成了 FunASR (Paraformer) 快速原型验证，后全部清除。核心问题：**验证方向不明确**——验证了 ASR 基本能力（中文能转写、流式能跑、热词能加），但没有回答"实际场景下到底能不能用"。

### 本次原则

**先验证场景，再优化引擎。** 不造 ASR，只选 ASR。先做语音转文字，意图理解后续再做。

## 共性前置验证：拾音质量

在所有验证之前，先确认大屏麦克风拾音是否可用。

### 场景特征

校长使用大屏时，通常是站在大屏前或坐在办公桌旁，距离麦克风 1-3 米。环境为办公室或会议室，可能有空调噪音。

### 方法

1. 在大屏前正常音量说话，录制 5 分钟
2. 距离 2-3 米处说话，录制 2 分钟
3. 回放听录音，主观评价：
   - 正常距离能否清晰拾取？
   - 远距离音量变化是否可接受？
   - 背景噪音是否淹没语音？
4. 用录音文件跑一次云API转写，看基础准确率

### 判定

- 可用 → 进入正式验证流程
- 不可用 → 需要外接麦克风或阵列，这是硬件问题不是 ASR 问题

## 验证一：三方云 API

### 目标

确认校长语音指令场景下转写是否可行，建立准确率基准线。

### 候选服务

| 服务商 | 流式 | 热词 | 备注 | 优先级 |
|--------|------|------|------|--------|
| 阿里云 | ✅ | ✅ | 同源 FunASR，生态近 | 第一批 |
| 讯飞 | ✅ | ✅ | 教育行业深耕 | 第一批 |
| 火山引擎 | ✅ | ✅ | 字节系 | 第二批 |
| 腾讯云 | ✅ | 有限 | - | 第二批 |
| 百度 | ✅ | 有限 | - | 第二批 |

### 前置：账号开通

```
阿里云
├─ 服务：智能语音交互 → 实时语音识别
├─ 协议：WebSocket
├─ 费用：新用户有免费额度，验证阶段够用
└─ 操作：阿里云控制台 → 智能语音服务 → 开通 → 获取 API Key

讯飞
├─ 服务：语音听写 / 实时语音转写
├─ 协议：WebSocket
├─ 费用：有免费试用额度
└─ 操作：讯飞开放平台 → 创建应用 → 开通语音听写 → 获取 API Key
```

### 测试数据

#### 录音清单

场景词汇特征：
- 数据指标：出勤率、合格率、平均分、升学率、及格率
- 组织结构：三年级、二年级一班、教务处、后勤处、德育处
- 人名地名：张老师、李校长、教学楼A栋、操场、食堂
- 动作指令：查看、导出、发送、对比、排序、筛选
- 时间范围：今天、本周、上个月、这学期

```
T01  安静·日常指令        3min
     "今天出勤率多少"
     "查看三年级的成绩"
     "把本周报表发给我"
     目的：基线准确率

T02  安静·数据指标        3min
     "本月各年级及格率对比"
     "上周校园安全事件统计"
     "这学期升学率趋势"
     目的：指标词汇准确率

T03  安静·组织人名        3min
     "二年级一班张老师的课表"
     "教务处李主任的待办"
     "教学楼A栋的教室使用情况"
     目地：专有名词准确率

T04  安静·快速指令        2min
     短句连续：每句 3-8 个字
     "看出勤""查成绩""导出报表""切换月视图"
     目的：短指令准确率和延迟

T05  噪音·日常指令        3min
     条件：开空调/风扇，正常音量
     内容同 T01
     目的：噪音影响

T06  远距离·日常指令      2min
     条件：离大屏 2-3 米
     内容同 T01 部分
     目的：拾音距离影响
```

录音要求：
- 格式：WAV / PCM，16kHz
- 每段开头说"第X段测试"便于对齐
- 可用手机录音制作

### 测试流程

```
第一步：快速筛选（1天）
│
│  用各云服务在线体验页面，播放测试音频
│  主观判断：基本准确率、延迟体感
│  ├── 明显不行 → 排除
│  └── 还行 → 进入第二步
│
第二步：精确测试（2-3天）
│
│  写简单测试工具接入 WebSocket API
│  跑 T01-T06 全部测试数据
│  记录量化指标
│  测热词：加入学校专有名词热词前后对比
│
第三步：出结论
   通过 → 进入验证二（自部署）
   失败 → 分析原因：拾音？噪音？引擎？
```

### 测试指标

| 指标 | 测量方法 | 通过标准 |
|------|---------|---------|
| 中文准确率 (CER) | 人工校对计算字错率 | 安静<5%，噪音<10% |
| 流式首字延迟 | 开始说话到第一个字出现 | <2s |
| 2pass 修正延迟 | partial 到 final 修正完成 | <3s |
| 热词提升 | 有/无热词准确率对比 | 专有名词 CER 降低 >30% |
| 短指令准确率 | T04 专项测试 | >95%（加热词后） |

### 测试矩阵

```
              T01    T02    T03    T04    T05    T06
              日常   指标   人名   短指令  噪音   远距离
阿里云-无热词   □      □      □      □      □      □
阿里云-有热词   □      □      □      □      □      □
讯飞-无热词     □      □      □      □      □      □
讯飞-有热词     □      □      □      □      □      □
```

### 通过/失败判定

- **通过**：至少一家云API在噪音环境下 CER < 10%（加热词）
- **失败**：所有云API噪音环境 CER > 15% → 问题在拾音或场景，不在引擎

## 验证二：自部署开源模型

### 前置条件

验证一通过。

### 目标

评估自部署能否接近云API效果，计算成本对比。

### 候选方案

| 方案 | 引擎 | 模型 | 流式 | 热词 | 部署方式 |
|------|------|------|------|------|---------|
| FunASR | Paraformer | paraformer-zh | ✅ 2pass | ✅ | Docker |
| FunASR | SenseVoice | sensevoice-zh | 待确认 | 待确认 | Docker |
| FunASR | 双模型 | 上述组合 | ✅ | ✅ | Docker |

### 测试方法

#### 1. 与云API使用相同测试数据

复用验证一的录音文件，保证可比性。

#### 2. 测试指标

| 指标 | 测量方法 | 通过标准 |
|------|---------|---------|
| 中文准确率 (CER) | 同验证一 | 与云API差距 <2% |
| 流式延迟 | 同验证一 | 首字<1.5s, final<3s |
| 热词效果 | 同验证一 | 与云API相当 |
| CPU 占用 | 服务器监控 | 单并发 <2核 |
| 内存占用 | 服务器监控 | <4GB |

#### 3. 成本对比模型

```
云API：      单价 × 日均调用量 × 30天
自部署：     服务器月租 + 运维人力（估算）

盈亏平衡点：  日均调用量达到 ___ 次时，自部署更便宜
```

### 通过/失败判定

- **通过**：自部署效果接近云API（CER 差距 <3%），且成本在可接受范围
- **失败**：效果差距大或运维成本不可接受 → 继续用云API

## 验证三：端侧部署

### 前置条件

验证一通过。验证二可选（端侧可独立验证可行性）。

### 目标

评估大屏本地运行 ASR 的可行性，作为离线兜底或降本方案。

### 候选方案

| 方案 | 引擎 | 模型 | 流式 | 热词 | 备注 |
|------|------|------|------|------|------|
| Sherpa-ONNX | Paraformer (量化) | 小模型 | ✅ | 待确认 | C++/ONNX, 适配Android |
| Sherpa-ONNX | SenseVoice (量化) | 小模型 | 待确认 | 待确认 | 需验证模型可用性 |
| PaddleLite | Paraformer | 轻量化 | ✅ | 待确认 | 百度移动端方案 |

### 硬件约束

```
大屏典型配置：
- SoC: ARM (瑞芯微/晶晨)
- RAM: 2-4GB (可用 ~1-3GB)
- 无独立 GPU
- 有麦克风（质量待测）
- 有网络
```

### 测试方法

#### 1. 性能基准

| 指标 | 测量方法 | 通过标准 |
|------|---------|---------|
| 推理实时率 (RTF) | 处理1s音频耗时 | <1.0 (即快于实时) |
| 内存占用 | 运行时 RSS | <1.5GB |
| 首字延迟 | 说话到出字 | <1s |
| 准确率 | 同验证一 | 安静环境 CER <15% |

#### 2. 量化模型效果对比

```
              完整模型   Int8量化   Int4量化
准确率          □         □         □
推理速度        □         □         □
内存占用        □         □         □
```

### 通过/失败判定

- **通过**：RTF < 1.0，安静环境 CER < 15%，内存 < 1.5GB → 可作为离线兜底
- **失败**：RTF > 1.0 或内存溢出 → 大屏算力不够，放弃端侧
