# 验证二：自部署开源模型 - 执行计划

## 目标

自部署 ASR 引擎，通过实时语音交互验证在学校管理场景下的转写效果，对比 Paraformer、SenseVoice、WeNet 和 Qwen3-ASR。

## 前置条件

- 本地 Mac 或 Linux 环境（Docker 可用）
- Qwen3-ASR 官方镜像通常需要 Linux + NVIDIA GPU；在 macOS 上主要用于接入验证脚本和 Demo
- 无需云API账号（自部署独立验证）

## Phase 1: 环境搭建

### 1.1 安装 Docker

```bash
# 检查 Docker 是否已安装
docker --version

# 未安装则安装 Docker Desktop
# Mac: https://docs.docker.com/desktop/install/mac-install/
# 安装后启动 Docker Desktop
```

### 1.2 启动 FunASR Docker

```bash
# 拉取 FunASR 服务端镜像
docker pull registry.cn-hangzhou.aliyuncs.com/funasr_repo/funasr:funasr-runtime-sdk-online-cpu-0.1.12

# 启动容器（CPU 版本）
docker run -p 10095:10095 -p 10096:10096 \
  registry.cn-hangzhou.aliyuncs.com/funasr_repo/funasr:funasr-runtime-sdk-online-cpu-0.1.12
```

启动后验证：

- `ws://localhost:10095` — 实时流式转写
- `ws://localhost:10096` — 离线转写

**功能需求**：

1. **连接管理**
   - 输入 WebSocket 地址
   - 连接/断开按钮
   - 连接状态显示
2. **音频输入**（两种方式）
   - **实时录音**：开始/停止录音，实时发送 PCM 音频到引擎
     - 区分显示 partial（实时）和 final（修正后）结果
   - **文件上传**：上传本地音频文件（wav/mp3/m4a 等），发送给引擎转写
     - 支持离线模式，说完再转、上传再转都适用
     - 适合用预录音频做可重复的对比测试
   - 结果自动滚动
3. **模式切换**
   - 2pass / online / offline 模式选择
   - 切换引擎地址（后续测 SenseVoice 时切换）
4. **热词管理**
   - 添加/删除热词
   - 发送热词配置给引擎
   - 预设学校管理场景热词
5. **结果管理**
   - 清空结果
   - 复制全部转写文本
   - 导出记录（便于后续对比分析）

**技术实现**：

```
┌────────────┐   PCM音频流/文件   ┌────────────┐
│  浏览器      │───────────────▶│  FunASR     │
│  index.html │◀──────────────│  Docker     │
│  原生 JS    │     转写结果     │  :10095     │
└────────────┘                └────────────┘

纯前端，无后端依赖
音频输入方式：
├── 实时录音：MediaRecorder API 采集 → PCM 流式发送
└── 文件上传：<input type="file"> → 读取 → 整段/分片发送
WebSocket 双向通信
```

## Phase 2: Paraformer 测试

### 2.1 测试方式

直接对着 Demo 说话，体验不同场景下的转写效果。

### 2.2 测试话术

每段说 3-5 句，说完观察转写结果：

```
A. 日常指令（基线）
   "今天出勤率多少"
   "查看三年级的成绩"
   "把本周报表发给我"
   "各班平均分排名"

B. 数据指标（专业词汇）
   "本月各年级及格率对比"
   "上周校园安全事件统计"
   "这学期升学率趋势"
   "教职工出勤率汇总"

C. 组织人名（专有名词）
   "二年级一班张老师的课表"
   "教务处李主任的待办"
   "教学楼A栋的教室使用情况"
   "德育处王老师的联系方式"

D. 快速短指令
   "看出勤"
   "查成绩"
   "导出报表"
   "切换月视图"
   "返回首页"

E. 噪音环境
   打开空调/风扇，重复 A 的话术

F. 远距离
   离麦克风 2-3 米，重复 A 的话术
```

### 2.8 通过判定

## Phase 3: SenseVoice 测试

### 3.1 部署方式调研（先做这个）

SenseVoice 部署方式待确认：

```
可能性A：FunASR 框架内直接支持
├── 替换模型配置即可
├── 操作：修改 FunASR Docker 启动参数
└── 最简单

可能性B：独立推理服务
├── 用 modelscope 下载模型
├── 用 Python 推理脚本运行
├── 需要写 WebSocket 适配层
└── 中等复杂度

可能性C：Sherpa-ONNX
├── 下载 ONNX 格式模型
├── 用 sherpa-onnx 推理
└── 和端侧方案打通，但集成工作量最大
```

调研步骤：

1. FunASR 官方文档是否支持 SenseVoice
2. SenseVoice GitHub 仓库的部署说明
3. Sherpa-ONNX 是否支持 SenseVoice
4. 确定最简部署路径后更新此计划

## Phase 4: WeNet 测试

### 4.1 部署与连接

使用 `wenet-e2e/wenet:latest` Docker 镜像，默认暴露 `ws://localhost:10097`。

验证步骤：

1. 启动 WeNet Docker 服务
2. 打开 WeNet Demo，连接 `ws://localhost:10097` 或本地代理
3. 测试实时录音是否有 partial / final 结果
4. 测试文件上传是否能返回最终结果

### 4.2 测试重点

- 复用 Paraformer 的 A-F 话术，保证三引擎可横向对比
- 开启和关闭热词，记录对专有名词和指标词的改善幅度
- 观察首字延迟、最终延迟和 final 修正幅度
- 记录 Docker 启动时间和资源占用

### 4.3 WeNet 特有关注点

- WebSocket 协议是否稳定，是否容易集成
- 热词偏置是否对学校场景关键词有效
- 流式识别体验是否优于 Paraformer

## Phase 5: Qwen3-ASR 测试

### 5.1 部署与连接

Qwen3-ASR 采用 `qwenllm/qwen3-asr` Docker 运行环境镜像，在容器内通过 `vllm serve Qwen/Qwen3-ASR-1.7B` 暴露 OpenAI 兼容接口：

- API base: `http://localhost:8001/v1`
- 转写接口: `POST /v1/audio/transcriptions`
- Demo 代理: `http://localhost:18083`

验证步骤：

1. 启动 Qwen3-ASR Docker 服务
2. 打开 Qwen3-ASR Demo，检测 `/v1/models`
3. 测试文件上传和录音后上传
4. 运行批量脚本，确认结果写入 `verification-2-self/qwen3-asr/results/`

### 5.2 测试重点

- 复用现有 A-F 话术和 manifest，保证四引擎横向可比
- 重点记录离线准确率、平均延迟、部署复杂度和显存/硬件要求
- 明确标注：当前接入路线不提供热词和流式 partial/final

### 5.3 Qwen3-ASR 特有关注点

- 官方镜像是否必须依赖 NVIDIA GPU
- `vllm serve` 的稳定性、冷启动时长和模型下载成本
- OpenAI 兼容接口是否便于后续统一接入更多多模态模型

## 风险与未知

| 风险                      | 影响     | 应对            |
| ----------------------- | ------ | ------------- |
| SenseVoice 不支持 FunASR   | 部署变复杂  | Phase 3 先调研   |
| WeNet Docker 镜像协议与预期不一致 | 集成变复杂 | 以 WebSocket 抓包和最小客户端先验证 |
| Qwen3-ASR 官方镜像只提供运行环境 | 集成变复杂 | 明确采用 `vllm serve` 路线并封装 start.sh |
| Qwen3-ASR 需要 NVIDIA GPU / Linux | 本机可能无法跑通 | 优先在 Linux GPU 机器验证，macOS 先完成接入 |
| Mac Docker 跑 FunASR / WeNet / Qwen3-ASR 性能差 | 延迟体感不准 | 改 Linux 服务器   |
| SenseVoice 无流式/无热词      | 不适合本场景 | 退回 Paraformer |
| Demo 开发耗时               | 推迟验证   | 保持功能最小化       |
