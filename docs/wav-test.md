# WAV 测试语料

本文档维护学校场景 ASR 测试语料的组织方式与覆盖原则。完整语料清单维护在 [config/audio-corpus.txt](/Users/lixiaofeng/Code/github/flowrium-voice/config/audio-corpus.txt)。

音频生成规范、版本规划和目录结构请参考 [docs/wav-generation-plan.md](/Users/lixiaofeng/Code/github/flowrium-voice/docs/wav-generation-plan.md)。

## 当前结构

当前语料共 `4` 类角色、每类 `60` 条，总计 `240` 条：

- 校长 `principal`
- 主任 `director`
- 老师 `teacher`
- 教职工 `staff`

每条语料采用轻量分层结构：

- `role`：角色
- `intent`：意图
- `variant_type`：表达类型，当前使用 `standard`、`colloquial`、`entity`、`short`
- `index`：角色内稳定编号
- `text`：原始测试文本

目录结构不变，仍然按 `version + role` 输出到 `audio/<version>/<role>/`；分层信息只进入 `manifest.csv`。

## 分层原则

每个意图固定保留 `6` 条表达：

- `2` 条标准表达：适合做基线识别率比较
- `2` 条口语表达：验证自然说法
- `1` 条实体表达：验证班级、人名、楼栋、时间等专有信息
- `1` 条短指令：验证低上下文命令

这种结构的目标不是把目录复杂化，而是让结果能按“角色 / 意图 / 表达类型”拆开分析，避免只有一个整体 CER。

## 角色覆盖

### 校长

覆盖 `10` 个高频意图：

- 运行总览
- 学生出勤
- 教师考勤与课时
- 教学质量与成绩
- 安全与违纪
- 设备与食堂
- 德育评价
- 招生与流失预警
- 家校沟通与教学任务
- 趋势分析与学校简报

### 主任

覆盖 `10` 个高频意图：

- 年级工作台
- 年级出勤与迟到
- 请假与预警
- 成绩分析
- 纪律与违纪
- 卫生与值班
- 调课与代课
- 教师管理与备课检查
- 重点学生与家长反馈
- 年级简报

### 老师

覆盖 `10` 个高频意图：

- 教学工作台
- 课表查询
- 班级名单与考勤
- 点名与调课
- 作业批改
- 成绩分析
- 课堂表现记录
- 表扬与学业预警
- 家校沟通与备课
- 课时与班级简报

### 教职工

覆盖 `10` 个高频意图：

- 办公工作台
- 待办与审批
- 请假申请
- 考勤与值班
- 通知公告
- 会议室预约
- 设备报修
- 食堂与服务进度
- 工资条
- 培训与通讯录

## 使用建议

- `20` 条级别的语料更适合冒烟验证和版本粗对比。
- 当前这套 `240` 条更适合对外演示、热词收益分析和按角色拆分评测。
- 如果后续要做鲁棒性专项，建议单独新增“打断、重说、停顿、纠错、噪音增强”语料，不要混进主评测集。
