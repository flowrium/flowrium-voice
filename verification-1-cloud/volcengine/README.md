# 火山引擎

## 厂商信息

火山引擎是字节体系的云与 AI 平台。对本次验证来说，它的价值不只是“能不能识别”，而是 `实时能力 + 大模型 ASR + 音视频链路复用` 这三件事放在一起看。

## 与本验证相关的 ASR 产品

- `豆包流式语音识别模型 2.0`：实时流式识别，适合字幕、语音交互、会议实时转写。
- `豆包录音文件识别模型 2.0`：异步文件识别，适合会议、课堂、质检。
- `大模型流式语音识别`：更偏高精度、上下文理解增强的大模型版本。
- `大模型录音文件识别`：标准版、极速版、闲时版，适合不同精度与时效需求。
- `豆包端到端实时语音大模型`：把 `ASR + LLM + TTS` 合在一起，适合未来实时对话场景。

## 支持的功能

- 实时流式识别、异步文件识别、极速文件识别。
- 新版控制台支持更简化的 `API Key + Resource ID` 鉴权。
- 流式大模型支持双向流式与流式输入两种模式，文档给出 `5 秒平均音频可在 300~400ms 返回` 的说明。
- 录音文件识别支持热词、标点、ITN、顺滑、说话人信息等能力。
- 大模型录音文件识别文档显示，默认支持 `中英文、上海话、闽南语、四川话、陕西话、粤语`，增强版还强调多语种和多格式能力。
- 豆包语音产品页强调 `多方言混合识别` 与个性化语音识别。

## 优势

- 价格非常激进，特别是非大模型版 `流式 0.9 元/小时、录音 0.75 元/小时`，很适合大规模试验。
- 实时链路强，若后续要接 RTC、实时互动、端到端语音对话，扩展性最好。
- 大模型版和普通版分层清楚，方便验证“精度提升值不值更高单价”。
- 新版 API 鉴权相对清爽，工程接入体验可能优于传统签名式方案。

## 劣势

- 产品迭代很快，型号、资源 ID、接口模式较多，选型和稳定落地需要多做一步梳理。
- 能力丰富但文档分散，第一次接入时理解成本略高。
- 非大模型版与大模型版能力、计费、默认并发不完全一致，容易混淆。

## 对校长语音指令场景的判断

- 如果你不仅关心识别准确率，还关心 `实时字幕 / 实时对话 / 后续多模态扩展`，火山引擎价值很高。
- 如果想做大规模样本对比，它是本轮里 `性价比最高` 的候选之一。
- 如果核心只是中文普通话识别基线，仍建议拿它与讯飞、阿里云并排实测，而不是只看价格。

## 重点验证项

- 非大模型版与大模型版在学校专有词、长句口语上的差异。
- 流式模式首字延迟、最终稳定结果与回改体验。
- 文件转写的说话人信息、标点和数字格式化是否足够好用。
- 接口结果结构是否便于直接进入后处理和评测链路。
