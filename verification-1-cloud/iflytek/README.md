# 讯飞开放平台

## 厂商信息

讯飞不是传统意义上的通用云厂商，而是国内语音能力最强势的专业平台之一。对中文、教育、会议、客服等场景的积累很深，因此非常适合放进本轮 `ASR 核心能力对照组`。

## 与本验证相关的 ASR 产品

- `实时语音听写`：短音频流识别，适合语音输入、人机交互，单次音频 `<= 60 秒`。
- `录音文件转写`：长音频异步转写，适合会议、访谈、质检，单文件 `<= 5 小时`。
- `极速录音转写大模型`：强调返回速度，官方给出 `1 小时音频约 20 秒转写完成` 的描述。
- `实时语音转写大模型`：不限时长流式识别，适合连续语音和实时字幕。
- `热词 / 个性化词表 / 垂直领域能力`：教育、医疗、金融、汽车等场景都有明显针对性。

## 支持的功能

- 短音频实时听写、长音频转写、极速转写、实时流式转写。
- 热词上传与个性化词表增强。
- 中文、英文、多语种及大量方言能力。
- 官方页显示：
  - `实时语音听写` 支持 `65 个语种、23 种方言、1 个民族语言`。
  - `录音文件转写/极速转写大模型` 支持 `中英 + 202 种方言免切`，并支持 `15 个多语种免切识别`。
- 支持角色分离、标点预测、数字/日期格式化。
- 支持 HTTP、WebSocket、SDK 与私有化方案。

## 优势

- 中文语音识别口碑强，教育、会议、客服等中文连续语音场景非常值得重点比。
- 方言与专有词能力强，适合学校、地区口音、教师姓名、课程名等复杂词汇。
- 录音文件转写与极速转写价格不高，做样本批量验证成本友好。
- 文档和产品页对场景划分清晰，适合按业务类型选型。

## 劣势

- 产品比较碎，`语音听写`、`录音转写`、`实时转写大模型`、`星火语音识别大模型` 之间要先选清楚。
- 同一品牌内不同产品的接口形态、计费单位和能力边界不完全一致。
- 如果希望后续把计算、对象存储、日志也放在同一家，讯飞不如阿里云、腾讯云方便。

## 对校长语音指令场景的判断

- 如果核心目标是验证 `中文识别效果上限`，讯飞应当进入 `首批验证`。
- 如果输入主要是普通话夹少量口语、方言、专有名词，讯飞理论上会是强候选。
- 如果后续要做完整云基础设施统一托管，讯飞更像 `能力组件型供应商`，而不是全栈云主方案。

## 重点验证项

- 中文连续说话时的断句稳定性和回改效果。
- 人名、校名、班级、日期、数字等专有词识别效果。
- WebSocket 实时接入复杂度与稳定性。
- 不同产品之间结果结构是否统一，是否利于后处理。
