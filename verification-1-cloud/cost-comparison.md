# ASR 成本对比

## 目的

这份文档单独从 `业务预算` 角度比较 5 家 ASR 厂商，不再停留在“官方目录价格”，而是直接换算成你关心的：

- `100 / 500 / 1000 / 5000` 用户规模
- 每人每天 `100` 条语音指令
- 每条语音指令平均 `5 秒`
- 分别看 `日成本 / 月成本 / 年成本`

## 统一假设

- 每个用户每天 `100` 条语音指令
- 每条平均 `5 秒`
- 每月按 `30 天`
- 每年按 `365 天`
- 只计算 `ASR 识别费用`
- 不含：
  - 存储
  - 网关服务
  - 日志
  - 热词训练
  - LLM
  - TTS
  - RTC / IM / 对话编排

## 厂商逐个判断

### 阿里云

- 优点是产品成熟、上云链路完整。
- 缺点是短语音指令场景价格不占优势。
- 更适合把 `ASR + OSS + 函数计算 + 日志` 放一起做统一方案。

### 讯飞开放平台

- 成本不算最低，但识别效果预期通常更强。
- 如果你更在意 `中文效果上限`，讯飞仍然值得优先测试。
- 更像“效果型选手”，不是“最低成本选手”。

### 火山引擎

- 当前这组估算里最便宜。
- 如果你的业务高频、短语音、实时化明显，火山引擎性价比很突出。
- 前提是接受用流式识别承接短指令。

### 腾讯云

- 成本中等，价格不极致，但比较稳。
- 如果后续还要接 `TRTC / COS / CLS`，工程整体体验较强。
- 更适合“成本尚可 + 工程成熟”的路线。

### 百度智能云

- 在短语音场景下价格很有竞争力。
- 如果还要叠加语义理解、内容处理，百度有一定延展空间。
- 属于“价格和能力比较均衡”的一档。

## 最适合你的场景的建议

### 如果目标是最低成本

### 如果目标是后续整套云上架构

### 如果目标是成本和能力均衡

- 备选：`讯飞开放平台`

## 注意事项

- 这份文档默认 `每条 5 秒`，如果你的真实平均时长变成 `10 秒`，火山引擎这类按小时计费的成本会大约翻倍。
- 按次计费的几家，对 `3 秒` 和 `8 秒` 的短语音差别通常不大，只要不超过单次时长限制即可。
- 真正上线时，还需要看：
  - 并发包
  - 商务折扣
  - 免费额度
  - 错误重试
  - 热词或自训练额外成本
