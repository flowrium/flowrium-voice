# 百度智能云

## 厂商信息

百度智能云属于 `AI 平台型云厂商`，在语音识别、语义理解和搜索相关能力上积累很早。放到本轮验证里，它比较像一个“老牌语音能力选手”，适合和讯飞、阿里云、腾讯云做横向对照。

## 与本验证相关的 ASR 产品

- `实时语音识别`：WebSocket 实时识别，适合字幕、会议、实时输入。
- `音频文件转写`：批量文件异步转写，适合会议、课堂、访谈、质检。
- `短语音识别标准版`：60 秒内短音频识别。
- `短语音识别极速版`：更强调速度，适合输入法、语音搜索、人机对话。
- `语音自训练平台`：上传词汇文本做领域优化。

## 支持的功能

- 实时流式识别、异步文件转写、短语音识别、极速短语音识别。
- 实时识别返回句子开始和结束时间。
- 音频文件转写支持自动区分发言人并带时间戳。
- 官方产品页强调普通话近场识别准确率可到 `98%`。
- 实时语音识别页强调支持中英文与部分方言识别。
- 音频文件转写页强调支持话者分离、VAD 时间戳与批量转写。
- 支持 `语音自训练平台`，官方描述可把垂直领域词汇识别率提升 `5%~20%`。

## 优势

- 语音与语义结合较深，数字格式化、文本纠错、语义相关处理能力一直是其强项。
- 实时识别和音频文件转写价格都不算高，适合做较完整的样本集验证。
- 话者分离与时间戳能力对会议、课堂录音特别有价值。
- 自训练平台对专有名词场景有潜力。

## 劣势

- 产品入口较多，官网有时通过页签切换展示价格，第一次查看容易混淆。
- 多语种和方言覆盖描述分散在不同页面，统一选型资料不如腾讯云、阿里云直观。
- 工程接入体验需要实测，不能只看产品页。

## 对校长语音指令场景的判断

- 如果你想重点看 `中文普通话 + 会议/课堂转写 + 说话人分离`，百度值得纳入第二梯队重点验证。
- 如果目标是实时短指令，它也具备竞争力，但更建议与讯飞、腾讯云直接跑同一批指令集。
- 如果后续还想做文本理解、摘要、内容分析，百度的 AI 平台联动也有延展空间。

## 重点验证项

- 实时识别在短指令和连续长句上的差异。
- 话者分离在老师/校长/学生多人说话样本中的可用性。
- 专有词、姓名、日期、班级编号等在自训练前后的提升效果。
- 接口文档、鉴权、SDK 与结果查询流程的工程复杂度。
