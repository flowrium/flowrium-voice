# 阿里云

## 厂商信息

阿里云是国内通用云厂商里产品线最完整的一类，适合把 `ASR + 存储 + 计算 + 日志` 放在同一云内统一落地。对本次验证最相关的是 `智能语音交互 ISI`。

## 与本验证相关的 ASR 产品

- `实时语音识别`：不限时长实时音频流，适合实时字幕、会议记录、语音输入。
- `录音文件识别`：异步文件转写，适合会议录音、质检、内容分析。
- `录音文件识别极速版`：更强调返回速度，适合短视频字幕、较高时效场景。
- `录音文件识别闲时版`：低价换时效，适合夜间批处理、低优先级转写。
- `语音自学习平台`：支持热词、语言模型优化，适合教育、政务、客服等垂直词汇场景。

## 支持的功能

- 实时流式识别与异步文件转写。
- 热词、自学习、领域语言模型优化。
- 时间戳输出，便于字幕或结果对齐。
- 多语种与多方言识别，官网页显示支持 `22 种方言、23 种语言`。
- 支持 API、SDK 与控制台开通。
- 并发可扩容，但超过默认额度需要单独购买并发包。

## 优势

- 通用云能力完整，后续若要接 `OSS / FC / ECS / SLS`，整体链路最顺。
- 价格结构清晰，既能小规模试，也能做大规模资源包采购。
- 产品形态全，实时、离线、极速、闲时版都能覆盖。
- 自学习平台成熟，适合有学校名、人名、课程名、班级名等热词优化需求的场景。

## 劣势

- 产品线较多，首次开通和选型要分清 `实时 / 录音 / 极速 / 闲时`。
- 若只做 ASR 能力验证，初期配置项和商业化路径稍重。
- 热词和自学习虽然有，但真实收益仍要靠样本实测，不能只看宣传。

## 对校长语音指令场景的判断

- 如果目标是先做一个国内通用基线，阿里云非常适合列为 `首批验证`。
- 如果未来希望语音、存储、计算、日志统一上云，阿里云是最自然的组合型方案之一。
- 如果更看重中文连续口语的极限识别效果，仍需要与讯飞、火山引擎做同样样本集对比。

## 重点验证项

- 学校名称、班级、日期、人数、课程等专有词识别表现。
- 实时接口首包延迟与最终结果稳定性。
- 文件识别和实时识别在断句、标点、数字格式上的差异。
- 热词和自学习对专有名词命中率的提升幅度。
