# 厂商对比表

## 横向对比

| 厂商 | 主要定位 | ASR 场景覆盖 | 接入方式 | 热词 / 自定义词 | 适合优先级 | 备注 |
| --- | --- | --- | --- | --- | --- | --- |
| 阿里云 | 通用云 + 语音能力 | 实时、离线 | HTTP / SDK | 倾向支持，需实测 | 高 | 适合作为国内通用云基线 |
| 讯飞开放平台 | 专注语音能力 | 实时、离线、连续语音 | HTTP / WebSocket / SDK | 倾向支持，需实测 | 高 | 中文语音能力值得重点验证 |
| 火山引擎 | 音视频 / AI / 数据 | 实时、离线 | HTTP / SDK | 待确认 | 中 | 适合看实时延迟和结果结构 |
| 腾讯云 | 通用云 + 音视频生态 | 实时、离线 | HTTP / SDK | 待确认 | 中高 | 国内接入成熟，便于组合部署 |
| 百度智能云 | AI 能力平台 | 短语音、实时、长语音 | HTTP / SDK | 倾向支持，需实测 | 中 | 适合作为国内 AI 厂商对照 |

## 建议验证顺序

1. 阿里云
2. 讯飞开放平台
3. 腾讯云
4. 火山引擎
5. 百度智能云

## 建议记录模板

每家都建议按同一批字段记录，避免最后只能做主观判断：

| 字段 | 说明 |
| --- | --- |
| 样本集 | 使用的音频文件集合 |
| 音频规格 | 采样率、声道、格式、时长 |
| 接口类型 | 实时流式 / 录音文件转写 |
| 识别结果 | 原文、厂商输出、差异点 |
| 关键词表现 | 校长、班级、课程、专有名词、数字 |
| 延迟 | 首包时间、整段完成时间 |
| 错误率 | CER / 关键词命中率 |
| 工程成本 | 鉴权、SDK、调试复杂度 |
| 费用 | 免费额度、单价、是否易超限 |
| 结论 | 是否进入下一轮 |

## 初步建议

- 如果目标是先快速找到国内可用基线，优先做 `阿里云` 和 `讯飞开放平台`。
- 如果目标是后续还要把存储、计算、日志也一起放在同一家，优先看 `阿里云` 或 `腾讯云`。
- 如果目标是突出中文语音识别本身，`讯飞开放平台` 和 `百度智能云` 更值得单独对比。
- 如果目标是后续还会扩展到音视频或实时互动，`火山引擎` 和 `腾讯云` 的参考价值更高。
